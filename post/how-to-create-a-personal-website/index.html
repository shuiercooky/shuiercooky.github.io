How to Create a Personal Website

1 Overview

If you want to have your own website, this blog will be your best helper. It will show you how to build and publish your website quickly from scratch.

2 Preparation for Construction

2.1 Documentation Preparation

There are two methods for you to choose:

  • Method 1 (Recommended) First, visit https://gohugo.io/ (Hugo official website). Then, click Quick Start on the homepage to get help information for building a website with Hugo.
  • Method 2
    Search other people’s building experience posts.

Method 2 is more direct, but method 1 will let you know more about hugo.

2.2 Software Package Preparation

Software Name Software Description Method of Obtaining Remark
Hugo Conveniently generate your personal website. https://github.com/gohugoio/hugo/releases Choose the software package suitable for personal computer.
Git Publish your website for others to visit. https://git-scm.com/download/win Choose the software package suitable for personal computer.
MARKDOWNPAD Edit blog content. https://markdownpad.com/download.html You can choose any convenient markdown editor.

3 Website Construction

3.1 PC Configuration

  1. Install Hugo, Git, and MARKDOWNPAD on your computer.
    Just follow the prompts to complete the installtion. Record the installation directory of Hugo and Git during installation.
  2. Add the installation path of Hugo and Git to the system environment variable Path.
    In the following, the Hugo path takes D:/hugo as an example, and the Git path takes D:/git as an example.
  3. Run cmd, execute hugo version and git version in sequence to confirm that hugo and git can be used normally.
  4. Create a subdirectory DocSite under D:/hugo.
    DocSite is used to store the website code.

3.2 Using Hugo to Generate Your Website

  1. Run cmd, exeucte cd D:/hugo/DocSite to enter your website path.
  2. Execute hugo new site Lee-DOC.github.io in the cmd window. （Lee-DOC.github.io is just an example.）
    The Lee-DOC.github.io folder is automatically generated in the D:/hugo/DocSite directory, which contains the relevant files of the website.
  3. Execute hugo new _index.md in the cmd window.
    _index.md is automatically generated in the D:/hugo/DocSite/Lee-DOC.github.io/content directory.
  4. Use MARKDOWNPAD to edit the _index.md file.
    Markdown tutorial can refer to https://www.runoob.com/markdown/md-advance.html.
  5. Download a theme that suits the style of your website from https://themes.gohugo.io/. Then unzip the downloaded theme package under D:/hugo/DocSite/Lee-DOC.github.io/themes. My choice is Techdoc. You can get it from https://github.com/thingsym/hugo-theme-techdoc.
  6. Configure the config.toml file. The default 3 parameters are as follows: (For more parameter settings, please refer to https://gohugo.io/getting-started/configuration/#all-variables-yaml.)
    • baseURL: According to Github Pages rules, the format is https://username.github.io/.
    • languageCode: Indicates the language.
    • title: Indicates the blog title.
  7. Execute hugo server -t hugo-theme-techdoc-master in the cmd window for local testing.
    hugo-theme-techdoc-master is the name of the theme stored in D:/hugo/DocSite/Lee-DOC.github.io/themes.
  8. After the server is started, visit http://localhost:1313 to visit the website, and if you find the problem, you can modify it in time.

3.2 Using Git to Publish Your Website

  1. Execute hugo -t hugo-theme-techdoc-master in the cmd window.
    Hugo will compile all files and output them to the D:/hugo/DocSite/public directory.
  2. (Optional) If you don’t have a Git account, just visit https://github.com/ and create your own account.
    My account name is shuiercooky.
  3. Create a repository on github with the name .github.io (for example, shuiercooky.github.io).
  4. (Optional) If it’s the first time for you to use Git, configure user information by referring to https://www.cnblogs.com/kinglead/p/10979641.html.
  5. Run cmd and exeucte cd D:/hugo/DocSite/public to enter the public directory.
  6. Execute the following commands in sequence:
    git init
    git remote set-url origin git@github.com:shuiercooky/shuiercooky.github.io.git
    git add .
    git commit -m “first commit”
    git push -u origin master
  7. Open <your user name>.github.io URL (for example, https://shuiercooky.github.io/), you can see your personal website.
